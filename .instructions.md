# Project Instructions

This repository is a React + TypeScript dashboard app with Tailwind CSS and localStorage-based persistence. When editing code here, follow the user’s explicit UI and behavior requests carefully.

## Key Rules

- Prioritize exact UI/UX changes requested by the user, especially in dashboard pages under `src/features/dashboard/*`.
- Preserve Mongolian label text and translate labels exactly as the user specifies.
- Avoid adding extra buttons, duplicate headings, or unnecessary UI elements unless explicitly requested.
- When the user asks for a modal flow change, implement the modal-only experience and remove alternative flows like bulk upload or extra dropdowns.
- Use the existing `localStorage` helper functions (`getLocalData`, `setLocalData`, `addLocalItem`, `updateLocalItem`, `deleteLocalItem`) for persistence and state synchronization.
- For notification behavior:
  - Ensure notifications are visible to all roles when requested.
  - Keep `CSR`, `Admin`, and `SuperAdmin` notification state in sync.
  - Implement `seenBy` logging and allow export of seen/unseen user reports when requested.
- If the user explicitly asks for confirmation before a destructive or send action, add a `window.confirm` prompt.

## Engineering Style

- Keep changes focused and minimal: update only the relevant component or behavior requested.
- Use Tailwind CSS classes consistent with the existing codebase.
- Maintain the current React hook and component style used in `src/features/dashboard/SuperAdminDashboard.tsx`, `AdminDashboard.tsx`, and `CsrDashboard.tsx`.
- Prefer existing helper functions and patterns over introducing new abstractions unless the request demands it.

## Clarifications

If a request is ambiguous, ask:
- Whether the rule should apply globally or only to specific dashboard components.
- Which user roles should be affected.
- Whether the request is a hard requirement or a preference.
